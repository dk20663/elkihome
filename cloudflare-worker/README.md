# Cloudflare Worker — Supabase-прокси для виджета

Зачем: домен `*.supabase.co` заблокирован Роскомнадзором. Без прокси
календарь на elkihome24.ru висит в РФ без VPN. Worker отдаёт те же
данные со своего (не заблокированного) домена.

Админ-приложение, синхронизация с Авито, база данных и edge-функции
**не меняются** — они продолжают работать напрямую с Supabase.

## Деплой за 5 минут (бесплатно)

### 1. Регистрация Cloudflare

1. https://dash.cloudflare.com/sign-up — зарегистрироваться (бесплатно).
2. После входа в левом меню: **Workers & Pages → Create → Create Worker**.
3. Имя: `elkihome-proxy` (или любое). Получится домен
   `https://elkihome-proxy.<ваш-логин>.workers.dev`.
4. Нажать **Deploy** (с дефолтным «Hello world»).

### 2. Загрузить наш код

1. После деплоя нажать **Edit code** (в правом верхнем углу).
2. В редакторе **полностью заменить** содержимое `worker.js`
   на код из файла [`worker.js`](./worker.js) в этой папке.
3. Нажать **Deploy** (правый верх).

### 3. Проверка

Открыть в браузере:

```
https://elkihome-proxy.<ваш-логин>.workers.dev/rest/v1/houses?select=*&apikey=<VITE_SUPABASE_PUBLISHABLE_KEY>
```

Должен прийти JSON со списком домов. Если да — прокси работает.

### 4. Подключение к виджету

Сообщи Lovable итоговый URL воркера (например
`https://elkihome-proxy.dk20663.workers.dev`). Lovable:

1. Пропишет URL в `vite.embed.config.ts` (константа `PROXY_URL`).
2. Пересоберёт `dist-embed/`.
3. Выпустит новый релиз `v2` в GitHub.

После этого в Тильде нужно поменять только `var VERSION = 'v1';` → `'v2';`.

## Лимиты бесплатного плана Cloudflare

- **100 000 запросов в день** — на виджет с трафиком elkihome24.ru
  хватит с запасом (одна загрузка календаря = ~5 запросов).
- Без ограничения по WebSocket-минутам (Realtime, если понадобится).

## Свой поддомен (опционально)

Если есть домен на Cloudflare (например `elkihome24.ru`):

1. В настройках воркера → **Triggers → Add Custom Domain**.
2. Указать, например, `api.elkihome24.ru`.
3. Сообщить Lovable новый URL — пересборка + новый релиз.

Свой поддомен надёжнее, чем `*.workers.dev` (Cloudflare-домены тоже
иногда страдают от точечных блокировок).

## Безопасность

Worker — прозрачный прокси. RLS-политики в Supabase **продолжают
действовать**: гостю по-прежнему доступны только публичные таблицы
(`houses`, `house_pricing`, `public_bookings_view`, `page_visits insert`).
Worker не добавляет привилегий и не хранит секреты.
